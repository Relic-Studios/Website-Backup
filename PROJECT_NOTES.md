## 📁 File Structure & How It Works

### Build System
- **Vite** is used for building the project
- `vite build` creates the `dist/` folder for deployment
- **IMPORTANT:** All assets must be in `public/` folder to be included in build
  - `public/assets/` → copied to `dist/assets/`
  - `public/js/` → copied to `dist/js/`
  - `public/css/` → copied to `dist/css/`

### Key Directories
```
Website/
├── index.html              # Main HTML file (DO NOT use type="module" on scripts)
├── vite.config.js          # Vite build configuration
├── package.json            # Dependencies
├── public/                 # Everything here gets copied to dist/
│   ├── assets/
│   │   ├── images/        # PNG, SVG images
│   │   ├── videos/        # MP4 video files (local, not Cloudinary)
│   │   ├── music/         # Audio files
│   │   └── models/        # 3D models
│   ├── js/                # JavaScript files
│   │   ├── landing-page.js
│   │   ├── main.js
│   │   ├── contact-form.js
│   │   ├── achievement-cards.js
│   │   └── (others)
│   └── css/               # CSS files
│       ├── landing-page.css
│       ├── style.css
│       ├── contact-form.css
│       └── (others)
├── netlify/
│   └── functions/
│       └── submit-contact.js  # Netlify serverless function for contact form
├── .netlifyignore         # Files to exclude from Netlify deployment
└── dist/                  # Build output (generated by `vite build`)
```

## 🔑 Important Lessons Learned

### 1. Vite Build Process
- Vite ONLY includes files from `public/` folder in the build
- If a file isn't in `public/`, it won't be in `dist/`
- Always check `dist/` folder before deploying

### 2. Asset Loading
- Don't block user entrance waiting for assets to load
- Assets can lazy-load after user enters the site
- Landing page should be fast and responsive

### 3. Video Hosting
- Cloudinary URLs need exact public_id with suffix
- Local videos are simpler and work reliably
- Videos are in `public/assets/videos/` and deploy with site

### 4. Netlify Deployment
- Build command: `vite build`
- Publish directory: `dist`
- Check deploy logs for "Uploading X files" to verify files changed
- "Uploading 0 files" means nothing changed

### 5. Form Handling
- Netlify Forms: Add `data-netlify="true"` to form tag
- Submit to `/` with `application/x-www-form-urlencoded`
- Automatic email notifications included
- Can also save to Neon database via Netlify Function

## 🗄️ Database Schema (Neon)

Table: `contact_submissions`

```sql
CREATE TABLE contact_submissions (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) NOT NULL,
    phone VARCHAR(50),
    company VARCHAR(255),
    event_type VARCHAR(100),
    event_date DATE,
    attendee_count VARCHAR(50),
    venue VARCHAR(255),
    budget VARCHAR(100),
    services JSONB,
    message TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Environment Variable:** `NETLIFY_DATABASE_URL` (set in Netlify dashboard)

## 🐛 Debugging Tips

### Landing Page Stuck
- Check browser console (F12)
- Verify `landing-page.js` is loaded
- Make sure button has `assetsReady = true`

### Videos Not Loading
- Check Network tab in DevTools
- Verify video files exist in `public/assets/videos/`
- Check paths in `main.js` portfolioConfig

### Contact Form Not Working
- Check Netlify Forms dashboard for submissions
- Verify `data-netlify="true"` on form
- Check Function logs in Netlify dashboard

### Build Issues
- Delete `dist/` folder and rebuild
- Check `public/` folder has all needed files
- Verify `.netlifyignore` isn't excluding needed files
